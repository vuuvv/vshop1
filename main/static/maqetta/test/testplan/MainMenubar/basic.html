Main menubar basic tests

New menu: New HTML File - default filename, no save

Steps

  1. Standard setup things: Clear the browser cache. Launch the daVinci server. Login if necessary. In some cases, you'll want to setup a virgin user environment by creating a whole new user login.
  2. From the New menu on the banner, choose New HTML File...
  3. Click OK to accept the default file name (file1.html)
  4. Drag/drop any widget onto canvas
  5. Close the editor by clicking "x" on editor's tab

Expected results

  1. After clicking OK on file dialog, editor window appears with a blank canvas, and...
  2. Files palette should show the file name for this file, and...
  3. Widget palette is populated with many widgets organized into categories (e.g., Form, Layout, etc), and...
  4. Outline palette contains a single entry (e.g., ./file1.html), and...
  5. Property palette appear but entries are disabled until a widget is selected, and...
  6. After closing, the file name should be removed from the Files palette.

New menu: New HTML File - default filename, with save

Steps

  1. From the New menu on the banner, choose New HTML File...
  2. Click OK to accept the default file name (file1.html)
  3. Drag/drop any widget onto canvas
  4. Click the "Save" button on editor toolbar
  5. Close the editor by clicking "x" on editor's tab
  6. Double-click on the filename in Files palette to reopen
  7. Close the editor by clicking "x" on editor's tab

Expected results

  1. After clicking OK on file dialog, ...(same as above)...
  2. After clicking save, dirty indicator (*) should disappear from editor's tab.
  3. (With 0.6, there is a known bug with dirty logic where you are prompted "are you sure" even though no changes since last save)
  4. After closing, the file name should continue to be visible in Files palette.
  5. After reopening, the visual page editor should show the widget you added to the page.

New menu: New HTML File - nested directory, with save

Steps

  1. From the New menu on the banner, choose New Folder...
  2. Type a name for the new folder or accept the default name. Keep the current parent folder as ".".
  3. Highlight the new folder name in the Files palette.
  4. From the New menu on the banner, choose New HTML File...
  5. You should see your folder name listed as "parent folder". Enter a filename and click OK.
  6. Drag a button widget onto the canvas.
  7. Click the "Save" button on editor toolbar
  8. Close the editor by clicking "x" on editor's tab
  9. Double-click on the filename in Files palette to reopen
  10. Close the editor by clicking "x" on editor's tab

Expected results

  1. A new editor window should appear (as with previous test sections).
  2. In the Files palette, the new file should appear as a nested file within the folder you just created..
  3. The button should be properly styled using the default theme (which in most cases is Claro, which results in bluish styling)

Settings Menu: Edit Preferences

Steps

  1. From the Settings menu on the banner, choose Edit Preferences...

Expected results

  1. As of now, there should be only one menu item on the Settings menu, Edit Preferences....
  2. Preferences dialog should appear. There should be a tree widget on the left with two major sections, JavaScript and HTML. Inside of the HTML, there should be a single entry, Visual Editor. The Visual Editor section should show various options, including Flow Layout, Snap to Grid, Show Grid and Warn on CSS Override.

Help Menu: daVinci Help and daVinci Community

Steps

  1. From the Help menu on the banner, choose daVinci Help
  2. From the Help menu on the banner, choose daVinci Community

Expected results

  1. As of now, there should be two one menu items on the Help menu, daVinci Help and daVinci Community.
  2. daVinci Help should cause a new browser tab to open showing the daVinci docs main page.
  3. daVinci Community should cause a new browser tab to open showing the daVinci connections community.
