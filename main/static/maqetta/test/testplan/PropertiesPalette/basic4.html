Properties palette basic tests (part 4)

CSS cascade features

Steps

  1. Create a virgin user environment navigating away from daVinci, clearing browser cache, killing the server, removing users/ folder, restarting the server, and navigating browser back to daVinci. (For server-hosted, create a new user)
  2. Issue a "New HTML File" command.
  3. NOTE: For much of this test, you will get prompts warning you that you might be changing a global CSS style sheet. Say OK to these warnings, but note that this will changes your theme CSS files. After running this test, you'll want to make sure you recreate your user so that the theme CSS files are restored to their original state.
  4. Drag a Button onto canvas.
  5. Go to Properties palette and go to 'width' property. Click on the right-pointing icon at the right side of the 'width' property to show the CSS cascade.
    You should see two entries, element.style and an entry that lists various CSS selectors that includes within it ".claro .dijitButton .dijitButtonNode".
    Up above in the type-in area of the 'width' property, type "200px". Click OK for the CSS global warning. (And similarly for other warnings you see later.)
    button width should change to 200px.
  6. Click on the show all rules checkbox.
    You should see additional entries in the CSS cascade list, including one whose selector is ".dijitButton".
    Click on the red "x" by the ".claro .dijitButton .dijitButtonNode" to remove the 'width' property declaration from that style rule.
    The button's width should no longer be 200px.
    Click on the radio button by the ".dijitButton" rule. Up above in the type-in area of the 'width' property, type "300px".
    The button's width should now be 300px, although this doesn't affect the actual button content, only the space it takes up in browser layout.
    Click on the radio button by the ".claro .dijitButton .dijitButtonNode" rule. Up above in the type-in area of the 'width' property, type "200px".
    The overall button width should still be 300px, but the button content area should now be 200px.
  7. Click on the "Save" icon on the main toolbar to save the new file. The asterisks that indicates that the file is "dirty" should go away.
  8. Click on the "X" close button for the editor tab for this file
  9. In the Files palette, double-click on the file you just created to re-open the file.

Expected results

  1. See text above for expected results.

app.css files

Steps

  1. Create a virgin user environment navigating away from daVinci, clearing browser cache, killing the server, removing users/ folder, restarting the server, and navigating browser back to daVinci. (For server-hosted, create a new user)
  2. Double-click on the app.css file in the Files palette.
  3. Add the following lines to the bottom of the file: 
    .myapp.claro .class1.dijitButton .dijitButtonNode {
	width:200px;
}
#label1 {
	font-size:30px;
}
  4. Save and close the file.
  5. Issue a "New HTML File" command.
  6. Drag two Buttons and two LABELs onto canvas.
  7. Select the second Button, go to Properties palette and set class to 'class1'.
  8. Select the second LABEL, go to Properties palette and set id to 'label1'.
  9. Click on the "Save" icon on the main toolbar to save the new file. The asterisks that indicates that the file is "dirty" should go away.
  10. Click on the "X" close button for the editor tab for this file
  11. In the Files palette, double-click on the file you just created to re-open the file.

Expected results

  1. Second button should have width of 200px and second LABEL should have font size of 30px.
